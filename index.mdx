Traccar is a production-ready, open-source GPS tracking server written in Java. It accepts connections from over 200 GPS protocols and 2,000+ device models, stores positions in any major SQL database, and exposes everything through a clean REST API. Deploy it yourself — on a single Linux server, in Docker, or in the cloud — and own your tracking data completely.

## What Traccar Does

Traccar listens on configurable TCP/UDP ports for incoming GPS data from tracking hardware. Each connection is decoded using a protocol-specific decoder, and the parsed position is stored in the database and forwarded to connected web clients in real time. A full REST API sits on top of the same data, so you can build integrations, dashboards, and automation without touching the internals.

## Get Started

<Steps>
  <Step title="Deploy the server">
    Use Docker Compose for the fastest path to a running server, or run the Linux installer for a bare-metal setup. See [Docker deployment](/deployment/docker) or [Linux installation](/deployment/linux).
  </Step>
  <Step title="Connect your first device">
    Find your device's protocol in the [protocol configuration guide](/configuration/protocols), open the required port in your firewall, and point the device at your server's IP address.
  </Step>
  <Step title="Explore the API">
    Authenticate with your admin credentials and call `GET /api/devices` to see your connected devices. The [API overview](/api/overview) covers authentication, pagination, and all available resources.
  </Step>
  <Step title="Set up notifications">
    Configure email or Telegram notifications so you receive alerts for geofence events, speeding, and alarms without polling the API. See the [notifications guide](/configuration/notifications).
  </Step>
</Steps>
