Traccar is a free, open-source GPS tracking platform designed for developers, fleet operators, and system integrators who need full control over their tracking infrastructure. Unlike hosted SaaS solutions, Traccar runs entirely on your own servers — giving you data sovereignty, unlimited devices, and complete freedom to extend or integrate the system. At its core, Traccar is a high-performance Java back-end that speaks to more than 200 GPS protocols and over 2,000 models of GPS tracking hardware, from low-cost consumer trackers to professional fleet telematics units.

## Architecture

Traccar is composed of three independently deployable layers that work together to deliver a complete tracking solution:

- **Traccar Server** — The Java-based back-end service in this repository. It listens on protocol-specific TCP/UDP ports for incoming device data, persists positions and events to a SQL database, and exposes a REST API on port 8082. This is the component you install and operate.
- **Traccar Web App** — A separate React-based front-end ([traccar/traccar-web](https://github.com/traccar/traccar-web)) that is served by the back-end server and provides the full management UI for devices, geofences, reports, and alerts. It ships bundled inside the server package.
- **Mobile Clients** — The [Traccar Client app](https://github.com/traccar/traccar-client) turns any Android or iOS device into a live GPS tracker. Third-party apps such as OsmAnd can also report positions using Traccar's built-in OsmAnd protocol on port 5055.

The server is stateless between requests and designed to scale from a single Raspberry Pi handling a handful of personal trackers up to a cloud VM managing thousands of concurrent fleet devices.

## Deployment Approaches

Traccar supports three primary deployment methods, ranging from zero-configuration quick-starts to fully customised production setups:

- **Docker** — The official `traccar/traccar` image bundles the server with a custom JRE and is the fastest path to a running instance. Pre-built Docker Compose files are provided for MySQL and TimescaleDB/PostgreSQL backends. See the [Docker deployment guide](/deployment/docker) for details.
- **Linux / Windows installer** — Official ZIP packages ship with a bundled JRE, so no Java installation is required on the target machine. A `setup.sh` script places files under `/opt/traccar` and registers a systemd service automatically. A Windows `.exe` installer registers Traccar as a Windows service.
- **Manual build** — Developers can clone the repository and build from source using Gradle. Refer to the [build documentation](https://www.traccar.org/build/) on the official website.

## Supported Databases

Traccar uses standard JDBC and works with all major SQL database engines:

| Database | Notes |
|---|---|
| **H2** | Embedded default. Zero configuration — ideal for evaluation and small deployments. Data stored in `./data/database`. |
| **MySQL** | Recommended for production. Full schema and Compose file provided. |
| **PostgreSQL** | Fully supported. TimescaleDB extension supported for time-series optimisation. |
| **MariaDB** | Compatible with the MySQL driver and configuration. |
| **Microsoft SQL Server** | Supported via JDBC. |

The active database is configured through the `database.driver`, `database.url`, `database.user`, and `database.password` keys in `traccar.xml`.
